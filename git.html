Git Revision 🟠

Git અને GitHub ના બધા કમાન્ડ્સ ગુજરાતીમાં!

1. Git શું છે? 🟠

Git એ એક Version Control System (VCS) છે. તે તમારા કોડમાં થતા દરેક ફેરફારને યાદ રાખે છે. જો ભવિષ્યમાં કંઈક લોચો થાય, તો તમે જૂના વર્ઝન પર પાછા જઈ શકો છો.
રીયલ લાઈફ: વિડીયો ગેમમાં 'Save Points'. તમે રમત રમો અને ગેમ સેવ કરો, જો આગળ જઈને મરી જાઓ (કોડ ક્રેશ થાય), તો તમે સેવ કરેલી જગ્યાએથી ફરી શરૂ કરી શકો!

શા માટે Git વાપરવું જોઈએ?

  • કોડની હિસ્ટ્રી સાચવવા માટે.
  • ટીમમાં સાથે કામ કરવા માટે.
  • અલગ-અલગ ફીચર્સ પર એકસાથે કામ કરવા માટે.
  • ભૂલ થાય તો પાછા જવા માટે.

2. Basic Git Commands 🛠️

પ્રોજેક્ટ શરૂ કરવાથી લઈને ફેરફારો સેવ કરવા સુધીના મુખ્ય કમાન્ડ્સ: 
# ૧. નવો પ્રોજેક્ટ શરૂ કરવો
git init

# ૨. ફાઈલનું સ્ટેટસ ચેક કરવું (કઈ ફાઈલ બદલાઈ છે)
git status

# ૩. ફાઈલને 'Staging Area' માં ઉમેરવી (તૈયારી કરવી)
git add file_name.js   # ફક્ત એક ફાઈલ
git add .              # બધી જ ફાઈલો

# ૪. ફેરફારોને 'કાયમી' સેવ કરવા (Commit)
git commit -m "તમારો મેસેજ અહીં લખો"

# ૫. અત્યાર સુધીના બધા જ કમિટ્સ જોવા
git log --oneline
એનાલોજી: ફોટો પડાવવો. - git add: બધા મિત્રોને ફોટોમાં વ્યવસ્થિત ગોઠવવા (તૈયારી). - git commit: કેમેરાનું બટન દબાવવું (ફોટો કાયમી સેવ થઈ ગયો).

3. Branching (બ્રાન્ચિંગ) 🌿

બ્રાન્ચિંગ એટલે મેઈન કોડને અસર કર્યા વગર નવા ફીચર પર અલગથી કામ કરવું. 
# ૧. બધી બ્રાન્ચ જોવી
git branch

# ૨. નવી બ્રાન્ચ બનાવવી
git branch feature-name

# ૩. બીજી બ્રાન્ચ પર જવું (સ્વિચ કરવું)
git checkout feature-name   # જૂની રીત
git switch feature-name     # નવી રીત

# ૪. નવી બ્રાન્ચ બનાવીને તરત ત્યાં જવું
git checkout -b new-feature
એનાલોજી: એક ઝાડની ડાળીઓ. મેઈન થડ (Main Branch) સુરક્ષિત રહે છે, તમે નવી ડાળી (Branch) પર નવું કામ કરી શકો છો. જો કામ સારું લાગે તો તેને થડ સાથે જોડી શકાય.

4. Remote Repositories (GitHub) 🌐

જ્યારે તમારે તમારો કોડ ઇન્ટરનેટ પર (GitHub, GitLab) મૂકવો હોય ત્યારે આ કમાન્ડ્સ વપરાય છે. 
# ૧. ઓનલાઈન રેપોઝીટરી જોડવી
git remote add origin https://github.com/user/repo.git

# ૨. કોડને ઓનલાઈન મોકલવો (Push)
git push origin main

# ૩. ઓનલાઈનથી નવો કોડ મેળવવો (Pull)
git pull origin main

# ૪. આખો પ્રોજેક્ટ કોપી કરીને ડાઉનલોડ કરવો
git clone https://github.com/user/repo.git
એનાલોજી: Google Drive. તમે તમારા કમ્પ્યુટરમાં ફાઈલ બનાવો અને પછી તેને ડ્રાઈવ પર અપલોડ (Push) કરો જેથી બીજા લોકો પણ જોઈ શકે.

5. Undo Changes (ફેરફારો પાછા ખેંચવા) ↩️

જો ભૂલથી કંઈક ખોટું લખાઈ જાય, તો તેને સુધારવાની રીતો: 
# ૧. ફાઈલમાં કરેલા ફેરફારો કાઢી નાખવા (પહેલા જેવી કરવી)
git checkout -- file_name.js

# ૨. સ્ટેજીંગ એરિયામાંથી ફાઈલ પાછી કાઢવી (Unstage)
git reset HEAD file_name.js

# ૩. છેલ્લો કમિટ રદ કરવો (પણ કોડ સાચવી રાખવો)
git reset --soft HEAD~1

# ૪. છેલ્લો કમિટ અને કોડ બંને પૂરેપૂરા ઉડાવી દેવા (સાવધાન!)
git reset --hard HEAD~1

# ૫. કોઈ જૂના કમિટને રિવર્સ કરવો (નવો કમિટ બનશે)
git revert commit_id

6. Stashing (કામચલાઉ સાચવવું) 📦

જ્યારે તમે કોઈ કામ કરતા હોવ અને અચાનક બીજી બ્રાન્ચ પર જવું પડે, ત્યારે અધૂરું કામ 'Stash' માં સાચવી શકાય છે. 
# ૧. અધૂરું કામ સાચવી લેવું (Temporary save)
git stash

# ૨. સાચવેલું કામ પાછું લાવવું
git stash pop

# ૩. સાચવેલા કામનું લિસ્ટ જોવું
git stash list
એનાલોજી: રસોઈ કરતી વખતે અચાનક કોઈ મહેમાન આવે, તો તમે શાક સમારવાનું બાજુ પર મૂકીને (Stash) મહેમાનને મળવા જાઓ. મહેમાન જાય એટલે ફરીથી શાક સમારવાનું શરૂ કરો (Pop).

7. Merging & Conflicts 🤝

જ્યારે એક બ્રાન્ચનું કામ બીજી બ્રાન્ચમાં ભેગું કરવું હોય ત્યારે Merge વપરાય છે. 
# ૧. મેઈન બ્રાન્ચમાં જવું
git checkout main

# ૨. બીજી બ્રાન્ચને મેઈન સાથે જોડવી
git merge feature-name

Merge Conflict શું છે?

જ્યારે બે લોકો એક જ ફાઈલની એક જ લાઈન પર અલગ-અલગ ફેરફાર કરે, ત્યારે Git મૂંઝાઈ જાય છે. આને Conflict કહેવાય. તમારે મેન્યુઅલી નક્કી કરવું પડે કે કયો કોડ રાખવો છે.

8. Git Best Practices ✅

  • નાના કમિટ્સ કરો: એકસાથે બહુ બધો કોડ કમિટ ના કરો.
  • સારો મેસેજ લખો: "Fixed bugs" ને બદલે "Login page validation error fixed" લખો.
  • મેઈન બ્રાન્ચ સાફ રાખો: હંમેશા નવી બ્રાન્ચ પર જ કામ કરો.
  • નિયમિત Pull કરો: બીજાના ફેરફારો લેતા રહો જેથી કન્ફ્લિક્ટ ના આવે.
  • .gitignore વાપરો: જે ફાઈલો (node_modules, .env) ઓનલાઈન ના મૂકવી હોય તેને આ ફાઈલમાં લખો.
